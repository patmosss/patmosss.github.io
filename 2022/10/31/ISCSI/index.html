    iSCSI

    iSCSI 服务端: 10.4.7.100 、iSCSI客户端: 10.4.7.101

    一、概述

    **互联网小型计算机系统接口**（**`iSCSI`**，`Internet Small Computer System Interface`）。`iSCSI` 是基于`TCP/IP` 网络的存储结构，实现数据的收发，简单地说，我们可以把它理解为网络存储。

传统的 `SCSI` 存储，是基于本地磁盘的，而 `ISCSI` 存储，是基于远程存储设备的。正因为有了网络的支持，使存储设备和应用服务器可以在空间上相互分离，并打破了本地存储受困于硬件端口数量的限制，在理论上可以实现无限扩展，也就是说在一台主机上用磁盘做了 `iscsi`，那么在另一台主机上是可以连接并使用的`iSCSI` 技术实现了物理硬盘设备与 `TCP/IP` 网络协议的相互结合，使得用户可以通过互联网方便地访问远程机房提供的共享存储资源。

    三、iSCSI 服务端

    执行 targetcli 命令进行其交互界面，使用 ls 命令查看 iSCSI 服务端的结构

    使用命令 cd 进入 /backstores/block，在该目录下使用 create 命令把 /dev/md5 加入共享资源池，并取名为 disk5。

    创建 iSCSI target 名称

    使用命令 cd 进入 iscsi 目录，使用命令 create 创建 iscsi target 名称，iscsi target 名称是由系统自动生成的一串用于描述共享资源的唯一字符串

    为 iSCSI target 配置共享资源

    系统在生成 target 名称的同时，还会在 /iscsi 目录中创建一个与其字符串同名的新“目录”用来存放共享资源。我们需要把前面加入到 iSCSI 共享资源池中的硬盘设备添加到这个新目录中。

    使用命令 cd 进入 luns 目录，使用命令 create 为 iscsi target 名称创建（添加）共享资源。

    设置 target 的访问控制列表

    iSCSI协议通过客户端名称进行验证，当iSCSI客户端设置的名称与服务端中设置的访问控制列表中某一名称条目一致即可，因此需要在iSCSI服务端的配置文件中写入一串能够验证用户信息的名称。

    设置 iSCSI 服务端的监听 IP 地址和端口号

    设置监听的IP地址和端口号，若出现（ Could not create NetworkPortal in configFS ）错误提示，可执行命令delete 0.0.0.0 3260后重试。

    设置完毕后，使用exit退出targetcli，并重启targetd服务。

    iscsiadm 是用于管理、查询、插入、更新或删除 iSCSI 数据库配置文件的命令行工具，用户需要先使用这个工具扫描发现远程 iSCSI 服务端，然后 查看找到的服务端上有哪些可用的共享存储资源。其中，

    • -m discovery 参数的目的是扫描并发 现可用的存储资源，
    • -t st 参数为执行扫描操作的类型，
